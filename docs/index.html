Neural Sentry

Active Defense Daemon

A Python IPS daemon that monitors Tor circuit creation in real time using python3-stem. When circuit creation spikes — a hallmark of DDoS or circuit flooding — it sends a NEWNYM signal to the Tor control port, forcing a circuit rotation that severs the attack vector instantly.

SSH Safety Valve

Cloud-Aware Ingress

The NFTables firewall defaults to dropping all inbound connections — including SSH. On Cloud VPS deployments, losing SSH access mid-install is fatal. The Safety Valve prompts interactively and surgically inserts a tcp dport 22 accept rule only when explicitly confirmed, leaving the hidden service fully isolated.

Balanced Firewall

NFTables v1.1 Stateful

Previous versions used aggressive rate limiting that strangled Tor's own directory fetches, causing "Onion Site Not Found" at launch. v10.0 uses ct state established,related accept — stateful tracking that allows Tor's outbound replies while dropping all unsolicited inbound. Port scans see nothing. Tor sees everything it needs.

Amnesic Logging

Volatile Memory Architecture

All Tor and sentry logs are mounted into a tmpfs RAM disk at /var/log/tor — 10MB, mode 0700. Power loss or reboot physically destroys all evidence. No data ever touches disk. The Privacy Monitor sanitizer also strips IPs from any log lines before they're written, enforcing SafeLogging 1 at both layers.

Traffic Obfuscator

Side-Channel Defence

A Lua script injected at the nginx body_filter_by_lua_file phase pads all HTTP responses to random block sizes and introduces 10–50ms timing jitter. This defeats traffic-correlation attacks that rely on consistent response sizes or timing signatures. The Privacy Monitor additionally strips Server, ETag, and X-Powered-By headers.

Anti-Forensic Architecture

Every layer is designed so that even a physical seizure of the hardware yields nothing useful to an adversary.

Response Manipulation

  • Response Padding: Lua body filter pads all responses to random block sizes, defeating content-length-based traffic correlation even inside encrypted Tor circuits.
  • Timing Jitter: 10–50ms random delay injected per request. Timing-correlation attacks that rely on consistent server latency are defeated at the response level.
  • Header Scrubbing: Server, ETag, and X-Powered-By headers removed. Fingerprinting nginx version or platform from response headers is impossible.
  • server_tokens off: nginx version string suppressed at config level as a second layer of version concealment.

Amnesic Storage

  • RAM-only logs: /var/log/tor mounted as tmpfs. All Tor notices and Neural Sentry events exist only in volatile memory.
  • Power-loss destruction: Cutting power or rebooting irrecoverably destroys all log data. No forensic recovery is possible from disk after shutdown.
  • IP sanitisation: privacy_log_sanitizer.py scrubs IPs and hostnames from log lines before they land in the RAM disk — even in-memory logs are pseudonymised.
  • AvoidDiskWrites: Tor configured with AvoidDiskWrites 1 to minimise state written to persistent storage.

Network Isolation & Identity Protection

Tor Configuration

  • HiddenServiceVersion 3 — v3 addresses are 56-character ed25519 keys, computationally infeasible to enumerate or spoof.
  • CookieAuthentication — control port requires cookie auth, no cleartext passwords.
  • Service binds to 127.0.0.1:80 only — nginx unreachable from any external interface.

Firewall Rules

  • Default inbound policy: DROP. No service reachable from the internet directly.
  • Loopback always accepted — Tor ↔ nginx communication over localhost never blocked.
  • Forward chain policy: DROP — no packet forwarding, preventing routing attacks.

Identity Management

  • Interactive identity wipe — type wipe at install to securely destroy old keys and get a fresh address.
  • SAVE_MY_ONION.sh backs up key material offline before any wipe.
  • nginx runs as www-data, Tor as debian-tor — full process isolation.

Frequently Asked Questions

Real questions from real deployments.

Does this work on Mobile (Termux / NetHunter)?

Yes. The Bare Metal installer targets Debian 13 Trixie and equivalent environments. It runs on any Debian-derivative that supports systemd, nftables, and tmpfs — including Termux (rooted Android) and Kali NetHunter.

Can I use this on a Cloud VPS (AWS / DigitalOcean)?

Yes — but you must enable the SSH Safety Valve during installation. The installer explicitly warns you and prompts. Saying No on a remote VPS will lock you out permanently once nftables rules apply. The Safety Valve inserts a tcp dport 22 accept rule without affecting Tor's circuit integrity.

Is it really log-free after a reboot?

Yes — with one caveat. The tmpfs mount is applied at install time but is not written to /etc/fstab. After a reboot, /var/log/tor reverts to a disk-backed directory. For persistent RAM-only logging across reboots, add a corresponding /etc/fstab entry or create a systemd mount unit.

What if I get an "Onion Site Not Found" error?

This is normal for brand-new v3 addresses. Tor must publish your hidden service descriptor to HSDir nodes — this takes 2–5 minutes. The Balanced Firewall ensures Tor's outbound directory traffic is not blocked. Check with systemctl status tor@default then retry in Tor Browser after a few minutes.

Why does nginx fail with a "unicode.mapping" error?

modsecurity.conf-recommended contains a SecUnicodeMapFile unicode.mapping 20127 directive using a relative path. ModSecurity resolves it relative to the conf directory — so the file must exist at /etc/nginx/modsec/unicode.mapping. v10.0 downloads it automatically and rewrites the directive to an absolute path. If on an older version, download it manually from the SpiderLabs/ModSecurity GitHub repo.

Why does Tor show "active (exited)" on Debian 13?

On Debian 13, tor.service is a dummy multi-instance master with ExecStart=/bin/true — exits immediately by design. The real Tor process is tor@default.service. Always check systemctl status tor@default. Aegis v10.0 detects this automatically.

Can I reinstall without losing my onion address?

Yes. When the installer detects an existing identity at /var/lib/tor/hidden_service/, it prompts you to type wipe or press Enter to keep it. Pressing Enter preserves your address through the reinstall. Use SAVE_MY_ONION.sh to back up key material before any wipe.

How do I update the website content?

Run sudo aegis-edit. It opens /var/www/onion_site/index.html in nano, then automatically fixes ownership and reloads nginx on save. For advanced edits, modify /var/www/onion_site/ directly then run systemctl reload nginx.
