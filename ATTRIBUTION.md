# Attribution Guidelines

When using, redistributing, or showcasing OnionSite-Aegis, please follow these attribution guidelines.

## Required Attribution

You **MUST** provide attribution to the original author when:
- Using OnionSite-Aegis in your projects
- Redistributing OnionSite-Aegis
- Showcasing or demonstrating OnionSite-Aegis
- Creating tutorials, videos, or documentation about OnionSite-Aegis
- Mentioning OnionSite-Aegis in blog posts, articles, or social media

## How to Give Attribution

### In Code/Documentation

Include a comment or notice like:

```bash
# OnionSite-Aegis - Privacy-Focused Tor Hidden Service Orchestrator
# Created by [Your Name/Author Name]
# https://github.com/[your-repo]/OnionSite-Aegis
```

### In README Files

Add a section like:

```markdown
## Credits

This project uses [OnionSite-Aegis](https://github.com/[repo]/OnionSite-Aegis) 
by [Author Name]. Shoutout to the creator for this amazing privacy-focused tool!
```

### In Videos/Tutorials

Verbally mention:
- "This is OnionSite-Aegis by [Author Name]"
- "Shoutout to [Author Name] for creating OnionSite-Aegis"
- "Credit to [Author Name] for this privacy tool"

And include in video description:
```
Tools used:
- OnionSite-Aegis by [Author Name] - https://github.com/[repo]/OnionSite-Aegis
```

### In Blog Posts/Articles

Include:
- A link to the original repository
- Credit to the author
- A mention like: "OnionSite-Aegis, created by [Author Name], is a..."

### In Social Media

When posting about OnionSite-Aegis:
- Tag or mention the author (if applicable)
- Include a link to the repository
- Use hashtags like #OnionSiteAegis #PrivacyTools
